Click any thumbnail — the photo opens fullscreen over a dimmed backdrop, and you drive the whole gallery from the keyboard.

A lightbox is the pattern you meet on every photo gallery, product page and image feed: a grid of thumbnails, and when you click one it expands into a fullscreen overlay — a fixed, dimmed, blurred layer with the full image centered on top. The interesting part is everything around the image. You can walk the gallery with the ← / → arrow keys or the on-screen buttons (they wrap at the ends), a caption + counter tells you it is image 3 / 8, clicking the image toggles zoom while clicking the backdrop closes, and Esc closes too. Under the hood it is done honestly: the page scroll locks so the background does not creep, Tab is trapped inside the dialog, and when you close, focus returns to the exact thumbnail you came from. The eight images below are self-contained SVG gradients — no external hosts. Open one and watch the read-out on the right update.

The layout is easy — a position:fixed layer with a centered image. The craft is the behaviour: scroll-lock so the page behind does not move, a focus trap so Tab cannot escape into the hidden page, keyboard nav (← → Esc), backdrop-vs-image click handling, and returning focus to the trigger so a keyboard user is not dumped at the top of the page. 🔨 Build it yourself

    
A lightbox is a fullscreen overlay that floats one image over a dimmed page, with prev/next, a counter, and a close. The layout is a position:fixed layer; the real work is the behaviour that makes it feel right and keeps it accessible: scroll-lock, keyboard nav, backdrop-vs-image clicks, a focus trap, and returning focus to the thumbnail. We build it over a gallery of eight self-contained SVG images, one piece at a time, no framework and no dependencies.

    
    
      
  1. 
        
    1
    The thumbnail grid (from data)
    
        
    Keep the gallery as an array of objects — a title, a category, and here a self-contained SVG gradient so the page has no external image hosts. Render each thumbnail as a real <button>, not a bare <img>: a button is keyboard-focusable and fires on Enter/Space for free, and it is the element we will return focus to on close. Give each one a data-i index and a clear aria-label.
    
        
    const GALLERY = [
  { title:"Alpine Sunrise", cat:"Landscape", w:1400, h:900,  grad:["#f59e0b","#ef4444"] },
  /* … 8 images, varied aspect ratios … */
];
grid.innerHTML = GALLERY.map((im,i) => `
  <button class="thumb" data-i="${i}"
          aria-label="Open ${im.title}, image ${i+1} of ${GALLERY.length}">
    <img src="${svgFor(im,i)}" alt="${im.title}" loading="lazy">
    <span class="thumb-cap">${im.title}</span>
  </button>`).join("");
    
      
    2. 
      
  2. 
        
    2
    The overlay: fixed, backdrop, centered image
    
        
    The overlay is one position:fixed element pinned to all four edges (inset:0) with a high z-index, a translucent dark background that dims the page, and an optional backdrop-filter:blur. Make it a flex container with align-items:center; justify-content:center so the image sits dead-center at any size, and cap the image with max-width/max-height in viewport units so it always fits. It starts hidden (display:none) and an .open class flips it to flex.
    
        
    .lb { position:fixed; inset:0; z-index:100; display:none;
      align-items:center; justify-content:center;      /* center the image */
      background:rgba(15,23,42,.92); backdrop-filter:blur(4px); }  /* the dim */
.lb.open { display:flex; }

.lb-img { max-width:90vw; max-height:76vh;   /* always fits the viewport */
          object-fit:contain; border-radius:12px; }
    
      
    3. 
      
  3. 
        
    3
    Open from a thumbnail — remember the trigger
    
        
    One delegated click listener on the grid catches every thumbnail (e.target.closest(".thumb")), so the wiring is one handler no matter how many images. open(i, trigger) stores the index, adds .open, and — critically — saves the element that opened it so we can return focus there later. Right now, capture lastTrigger; steps 9–10 spend it. Then move focus into the dialog so the keyboard is immediately inside it.
    
        
    let idx = 0, opened = false, lastTrigger = null;

grid.addEventListener("click", e => {
  const b = e.target.closest(".thumb");
  if (b) open(+b.dataset.i, b);          // pass the button as the trigger
});

function open(i, trigger){
  idx = i;
  lastTrigger = trigger || document.activeElement;   // remember for return-focus
  lb.classList.add("open");
  render();                              // fill src / caption / counter
  lbClose.focus();                       // move focus INTO the dialog
  opened = true;
}
    
      
    4. 
      
  4. 
        
    4
    Lock the page scroll while open
    
        
    If you do nothing, scrolling with the lightbox open scrolls the page behind it — the background creeps, which feels broken. The fix is one line: set document.body.style.overflow = "hidden" on open and restore it ("") on close. That freezes the page. (On desktop, hiding the scrollbar can shift layout by its width; the polished version measures window.innerWidth - documentElement.clientWidth and adds it as right padding so nothing jumps — but the one-liner is the core idea.)
    
        
    function open(i, trigger){
  /* … */
  document.body.style.overflow = "hidden";   // SCROLL-LOCK the page
}
function close(){
  /* … */
  document.body.style.overflow = "";          // restore scrolling
}
/* optional: avoid the scrollbar-width layout jump */
const sw = window.innerWidth - document.documentElement.clientWidth;
document.body.style.paddingRight = sw + "px";
    
      
    5. 
      
  5. 
        
    5
    Prev / next with wraparound + counter
    
        
    Navigation is one function. go(step) moves the index by +1 or -1, and the modulo trick (idx + step + n) % n makes it wrap — past the last image you land back on the first, and before the first you land on the last, with no if branches. Adding + n before the modulo keeps the result non-negative when stepping back from index 0. After moving, render() swaps the src, updates the caption, and rewrites the 3 / 8 counter so the viewer always knows where they are.
    
        
    function go(step){
  const n = GALLERY.length;
  idx = (idx + step + n) % n;   // +n so -1 from index 0 wraps to the last
  render();
}
lbPrev.onclick = () => go(-1);
lbNext.onclick = () => go(+1);

function render(){
  const im = GALLERY[idx];
  lbImg.src = svgFor(im, idx);  lbImg.alt = im.title;
  lbTitle.textContent   = im.title + " · " + im.cat;
  lbCounter.textContent = (idx + 1) + " / " + n;   // the counter
}
    
      
    6. 
      
  6. 
        
    6
    Keyboard navigation (← → Esc)
    
        
    A lightbox that only works with the mouse is half a lightbox. Attach one keydown listener that only acts while open: ← is go(-1), → is go(1), and Esc closes. Call preventDefault() so the arrows do not also scroll the page underneath. This is the same go() and close() the buttons use — one behaviour, two ways in — so the mouse and keyboard can never drift out of sync.
    
        
    document.addEventListener("keydown", e => {
  if (!opened) return;                 // ignore keys unless the lightbox is up
  if (e.key === "Escape")     { e.preventDefault(); close(); }
  else if (e.key === "ArrowLeft")  { e.preventDefault(); go(-1); }
  else if (e.key === "ArrowRight") { e.preventDefault(); go(1); }
  else if (e.key === "Tab")   { trapTab(e); }   // step 9
});
    
      
    7. 
      
  7. 
        
    7
    Click backdrop to close vs click image
    
        
    People expect clicking the dark area around the image to close, but clicking the image itself should not close. The clean way is to check where the click landed: if e.target is the overlay element itself, the click was on the backdrop, so close; a click on the image bubbles up with e.target === lbImg, so we handle that separately (here, toggle zoom) and stop it from reaching the overlay. No invisible catch-layers, no guessing coordinates — just identity checks on the event target.
    
        
    // clicking the overlay ITSELF (the backdrop) closes
lb.addEventListener("click", e => { if (e.target === lb) close(); });

// clicking the IMAGE does not close — it toggles zoom, and we stop
// the event so it never reaches the backdrop handler above
lbImg.addEventListener("click", e => { e.stopPropagation(); toggleZoom(); });

  8. 
        
    8
    The zoom toggle
    
        
    A nice extra: click the image (or a zoom button) to blow it up. Keep it as pure state — a zoomed boolean toggling a .zoomed class that applies a CSS transform:scale(). Reset zoom to off every time you navigate or close, so you never open the next image already zoomed. Swap the cursor (zoom-in ⇄ zoom-out) and flip aria-pressed on the button so the control announces its state.
    
        
    let zoomed = false;
function setZoom(on){
  zoomed = on;
  lbImg.classList.toggle("zoomed", on);       // CSS: transform:scale(1.7)
  lbZoom.setAttribute("aria-pressed", on);
  lbZoom.textContent = on ? "Zoom out" : "Zoom in";
}
function toggleZoom(){ setZoom(!zoomed); }
// render() and close() both call setZoom(false) -> never carry zoom over
.lb-img.zoomed { transform:scale(1.7); cursor:zoom-out; }

  9. 
        
    9
    Trap focus inside the dialog
    
        
    While the overlay is open, the page behind it is inert — but the browser does not know that, so Tab would happily walk into the hidden thumbnails. A focus trap keeps Tab cycling among the dialog controls only. Collect the focusable elements (the close, prev, next and zoom buttons), and when Tab hits the last one, loop back to the first; when Shift+Tab hits the first, loop to the last. Combined with moving focus in on open (step 3), the keyboard can never escape into the page underneath.
    
        
    function trapTab(e){
  const f = lb.querySelectorAll("button:not([disabled])");
  if (!f.length) return;
  const first = f[0], last = f[f.length - 1];
  if (e.shiftKey && document.activeElement === first){
    e.preventDefault(); last.focus();        // Shift+Tab off the first -> last
  } else if (!e.shiftKey && document.activeElement === last){
    e.preventDefault(); first.focus();       // Tab off the last -> first
  }
}

  10. 
        
    10
    ARIA + return focus on close
    
        
    The honest finish. Mark the overlay role="dialog" with aria-modal="true" and an aria-label so assistive tech announces a modal image viewer. Give every button a real aria-label, and use a visually-hidden aria-live region to speak each change (Alpine Sunrise, image 1 of 8). Most important: on close(), call lastTrigger.focus() to send focus back to the thumbnail the user opened from — without it, focus falls to the top of the document and a keyboard user is lost. That single line is the difference between a demo and a real component.
    
        
    <div class="lb" role="dialog" aria-modal="true" aria-label="Image viewer">…</div>

function close(){
  lb.classList.remove("open");
  document.body.style.overflow = "";
  setZoom(false);
  opened = false;
  if (lastTrigger) lastTrigger.focus();   // RETURN focus to the trigger
  announce("Viewer closed.");
}
function announce(msg){ live.textContent = msg; }   // one aria-live region
