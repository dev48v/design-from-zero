💀 Skeleton Loaders — Shimmer While You Wait

Watch the gray blocks become real content.

Hit reload and the card snaps back to shimmer placeholders — gray blocks shaped exactly like the avatar, title, lines, and image — for about 1.5s, then the real content fades in. Because the skeleton mirrors the final layout, nothing jumps. Flip on side-by-side to compare skeleton vs loaded, and tweak the shimmer speed & direction.

What's happening under the hood

Each gray block is a .skel div sized to match its real element. The shimmer is a single light-colored linear-gradient in a ::after layer that slides across with a transform:translateX keyframe. On reload we show the skeleton, wait ~1.5s for "data", then swap in the real markup with a fade — same dimensions, so the layout never shifts.

The whole idea: a skeleton mirrors the final layout's shape so the page doesn't jump (no layout shift) and feels faster than a spinner — the shimmer is just an animated gradient sweeping across gray blocks.

🔨 Build it yourself

    
Gray blocks shaped like your content, one shimmer gradient, and a timed swap. Pure CSS plus a few lines of JS, no libraries.

    
    

      
  1. 1
    The skeleton markup — blocks shaped like the real card
    
        
    <!-- mirror the final layout: avatar + title + lines + image -->
<div class="card">
  <div class="row">
    <div class="skel avatar"></div>        <!-- round -->
    <div>
      <div class="skel line title"></div>   <!-- name -->
      <div class="skel line sub"></div>     <!-- handle -->
    </div>
  </div>
  <div class="skel line"></div>             <!-- body -->
  <div class="skel line short"></div>
  <div class="skel image"></div>            <!-- media -->
</div>

  2. 2
    The base block — a rounded gray div
    
        
    /* every placeholder shares this look */
.skel {
  position: relative;     /* anchor for the shimmer layer */
  overflow: hidden;       /* clip the moving gradient */
  background: #e2e8f0;    /* a calm light-gray */
  border-radius: 8px;
}
/* a generic text line */
.line   { height: 12px; margin: 8px 0; }
.short  { width: 60%; }
.avatar { width: 48px; height: 48px; border-radius: 9999px; }
.image  { height: 180px; border-radius: 12px; }
    
      
    3. 

      
  3. 3
    The shimmer — an animated gradient that sweeps across
    
        
    /* a light band riding in an ::after layer */
.skel::after {
  content: "";
  position: absolute; inset: 0;
  transform: translateX(-100%);          /* start off the left edge */
  background: linear-gradient(90deg,
    rgba(255,255,255,0)   0%,
    rgba(255,255,255,.65) 50%,           /* the highlight */
    rgba(255,255,255,0)   100%);
  animation: shimmer 1.4s ease-in-out infinite;
}
@keyframes shimmer {
  100% { transform: translateX(100%); }  /* glide off the right edge */
}
    
      
    4. 

      
  4. 4
    Reserve the real dimensions (no layout shift)
    
        
    /* The skeleton must occupy the SAME box as the real element.
   Match width + height so when content arrives, nothing moves. */
.image      { height: 180px; }   /* real photo is also ~180px */
.avatar     { width: 48px; height: 48px; }
.title      { width: 40%; height: 14px; }

/* Tip: if the real text wraps to 3 lines, draw 3 skeleton lines.
   A matching shape = a Cumulative Layout Shift (CLS) of ~0. */
    
      
    5. 

      
  5. 5
    The swap — skeleton while loading, real content on ready
    
        
    const slot = document.querySelector("#feed");

// 1) show the skeleton immediately
slot.innerHTML = skeletonHTML;
slot.setAttribute("aria-busy", "true");   // tell AT it's loading

// 2) fetch the real data
const data = await fetch("/api/post").then(r => r.json());

// 3) replace with real markup (fades in via .real-in)
slot.innerHTML = realHTML(data);
slot.removeAttribute("aria-busy");
    
      
    6. 

      
  6. 6
    Fade the real content in
    
        
    /* a gentle entrance so the swap doesn't snap */
.real-in { animation: fadeIn .5s ease-out; }
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(8px); }
  to   { opacity: 1; transform: none; }
}
/* wrap the real markup so it animates on insert:
   realHTML = `<div class="real-in">...content...</div>` */
    
      
    7. 

      
  7. 7
    Respect prefers-reduced-motion
    
        
    /* some users get motion sick — kill the sweep for them.
   Leave a static, slightly dimmed gray so it still reads
   as "loading", just without the moving highlight. */
@media (prefers-reduced-motion: reduce) {
  .skel::after { animation: none; opacity: .5; }
}
    
      
    8. 

      
  8. 8
    Tune speed & direction with a CSS variable
    
        
    /* drive duration from a variable so JS can tweak it live */
.skel::after { animation: shimmer var(--shimmer-dur, 1.4s) ease-in-out infinite; }
.skel.rev::after { animation-direction: reverse; }  /* right-to-left */

// JS: a speed slider + direction toggle
card.style.setProperty("--shimmer-dur", speed + "s");
card.querySelectorAll(".skel")
    .forEach(s => s.classList.toggle("rev", dir === "rtl"));
