🎚 Range Slider — Dual-Thumb Min/Max, No-Cross Clamp + ARIA Slider

Drag either thumb — or focus one and press ← → Home End.

This is a real dual-thumb range slider, built from scratch in vanilla JS — no <input type="range">, no library. Two thumbs slide over a track; the indigo fill between them shows the chosen range; the thumbs cannot cross (the low value is clamped to stay ≤ the high value). Drag works with mouse, pen and touch through a single set of pointer events; clicking anywhere on the track jumps the nearest thumb there and lets you keep dragging. Each thumb is a proper role="slider" with live aria-valuemin/max/now/text, and takes the full keyboard: ←/→ step, Shift+arrow (or PageUp/PageDown) jump a big step, Home/End snap to the ends. Every value is snapped to the step, and positions are computed by mapping value ⇄ pixel.

Range doesn't have to start at zero — value ⇄ pixel mapping is (v − min) / (max − min), so any numeric window works: prices, years, temperatures, ratings.

Each thumb is role="slider" with aria-valuemin, aria-valuemax, aria-valuenow and a formatted aria-valuetext. In the dual slider the two thumbs constrain each other: the low thumb's aria-valuemax is the current high value and the high thumb's aria-valuemin is the current low value — the exact WAI-ARIA multi-thumb pattern used by price filters on Amazon, Airbnb and Booking.com.

🔨 Build it yourself

    
A track, an indigo fill, one thumb per handle, a two-way value ⇄ pixel map, step snapping, a no-cross clamp so the low value never passes the high value, unified pointer drag that also covers touch, click-on-track to move the nearest thumb, a full keyboard per thumb, and live ARIA slider state. ~80 lines of vanilla JS — no framework, no dependencies, and no <input type="range">.

    
    
      
  1. 
        
    1
    Markup: track, fill, and one thumb per handle
    
        
    The track holds an absolutely-positioned fill and one role="slider" thumb per handle. Each thumb is focusable (tabindex="0"), carries an aria-label, and declares its value range up front. For a single-value slider you keep just one thumb — everything else is identical.
    
        
    <div class="rs">
  <div class="rs-track">
    <div class="rs-fill"></div>
    <div class="rs-thumb" role="slider" tabindex="0"
         aria-label="Minimum price"
         aria-valuemin="0" aria-valuemax="300" aria-valuenow="40"></div>
    <div class="rs-thumb" role="slider" tabindex="0"
         aria-label="Maximum price"
         aria-valuemin="40" aria-valuemax="400" aria-valuenow="300"></div>
  </div>
</div>
    
      
    2. 
      
  2. 
        
    2
    CSS: position by percent, not pixels
    
        
    The track is position:relative; the fill and thumbs are positioned with a percentage left. Percentages make the slider resize-proof — no resize listener needed, because 40% of the track is always 40% wide. Centre each thumb over its value with margin-left:-10px (half its width). Set touch-action:none so a drag on touch doesn't scroll the page.
    
        
    .rs-track{ position:relative; height:6px; background:#e0e7ff; border-radius:999px; }
.rs-fill { position:absolute; top:0; height:100%; background:#4f46e5;
           border-radius:999px; pointer-events:none; }
.rs-thumb{ position:absolute; top:50%; width:20px; height:20px; margin-left:-10px;
           transform:translateY(-50%); background:#fff; border:2px solid #4f46e5;
           border-radius:50%; cursor:grab; touch-action:none; }
.rs-thumb:focus-visible{ box-shadow:0 0 0 4px rgba(99,102,241,.4); }
    
      
    3. 
      
  3. 
        
    3
    value ⇄ pixel — the two-way map
    
        
    The whole slider is one coordinate transform. pct(v) turns a value into a percentage for rendering. valueAt(clientX) goes the other way: measure the track with getBoundingClientRect(), turn the pointer's x into a 0→1 ratio, and map it back into the value range. Clamp the ratio so pointers outside the track still land on a valid end.
    
        
    const clamp = (v,a,b) => Math.min(b, Math.max(a, v));
const pct   = v => ((v - min) / (max - min)) * 100;      // value → %

function valueAt(clientX){                                // pixel → value
  const r = track.getBoundingClientRect();
  let ratio = clamp((clientX - r.left) / r.width, 0, 1);
  return snap(min + ratio * (max - min));
}
    
      
    4. 
      
  4. 
        
    4
    Snap to the step (and beat float error)
    
        
    A slider with step only lands on multiples of that step from min. Round to the nearest step, then re-clamp into range. Floating-point steps like 0.5 produce ugly tails (2.7500000001), so round to the step's own decimal count with toFixed before returning.
    
        
    const decimals = (String(step).split(".")[1] || "").length;
function snap(v){
  const s = Math.round((v - min) / step) * step + min;    // nearest step
  return clamp(parseFloat(s.toFixed(decimals)), min, max); // kill float tail
}
    
      
    5. 
      
  5. 
        
    5
    setVal — the no-cross clamp + render
    
        
    One function moves a thumb. Before committing, enforce that the thumbs can't cross: the low thumb is clamped to [min, currentHigh] and the high thumb to [currentLow, max]. Then render — set the fill and thumb percentages and refresh every ARIA attribute, including each thumb's constrained aria-valuemin/max so screen readers know the live limits.
    
        
    function setVal(i, v){
  v = snap(v);
  if (dual){
    if (i === 0) v = clamp(v, min, vals[1]);   // low ≤ high
    else         v = clamp(v, vals[0], max);   // high ≥ low
  }
  vals[i] = v;
  render();                                    // %-position fill + thumbs
  onChange && onChange(dual ? {lo:vals[0], hi:vals[1]} : {value:vals[0]}, i);
}
    
      
    6. 
      
  6. 
        
    6
    Drag with pointer events (mouse + touch + pen)
    
        
    Pointer events give you one code path for mouse, touch and pen. On pointerdown, call setPointerCapture so this thumb keeps receiving moves even when the finger slips off it; then each pointermove maps x → value → setVal. Because we captured, the matching pointerup always fires on the same thumb, so cleanup is reliable.
    
        
    thumb.addEventListener("pointerdown", e => {
  e.preventDefault();
  thumb.setPointerCapture(e.pointerId);   // keep events even off-thumb
  thumb.classList.add("dragging");
  dragging = i; thumb.focus();
});
thumb.addEventListener("pointermove", e => {
  if (dragging === i) setVal(i, valueAt(e.clientX));
});
thumb.addEventListener("pointerup", () => {
  dragging = -1; thumb.classList.remove("dragging");
});
    
      
    7. 
      
  7. 
        
    7
    Click the track → move the nearest thumb
    
        
    A press on the track itself (not a thumb) should grab whichever thumb is closest to the click and then let you keep dragging it. Work out the value at the click, pick the thumb with the smallest distance, move it, and hand the pointer to that thumb with setPointerCapture so the same gesture becomes a drag.
    
        
    track.addEventListener("pointerdown", e => {
  if (e.target.closest(".rs-thumb")) return;   // thumb handles its own
  const v = valueAt(e.clientX);
  const i = (!dual || Math.abs(v - vals[0]) <= Math.abs(v - vals[1])) ? 0 : 1;
  setVal(i, v);
  thumbs[i].setPointerCapture(e.pointerId);    // continue as a drag
  dragging = i; thumbs[i].classList.add("dragging"); thumbs[i].focus();
});
    
      
    8. 
      
  8. 
        
    8
    Keyboard per thumb + when to reach for it
    
        
    Each focused thumb takes the standard slider keys: arrows nudge by step, Shift+arrow / PageUp/PageDown jump a big step, Home/End snap to that thumb's own limits (so the no-cross rule holds for free). preventDefault() stops the page scrolling. Use a range slider for picking a value or window from a continuous numeric range where the rough position matters more than an exact figure — price, volume, dates. If people need a precise number, pair it with (or fall back to) a number input.
    
        
    thumb.addEventListener("keydown", e => {
  const big = e.shiftKey ? bigStep : step, v = vals[i]; let n = v, hit = true;
  switch (e.key){
    case "ArrowRight": case "ArrowUp":   n = v + (e.shiftKey ? bigStep : step); break;
    case "ArrowLeft":  case "ArrowDown": n = v - (e.shiftKey ? bigStep : step); break;
    case "PageUp":   n = v + bigStep; break;
    case "PageDown": n = v - bigStep; break;
    case "Home": n = (dual && i === 1) ? vals[0] : min; break;  // own lower limit
    case "End":  n = (dual && i === 0) ? vals[1] : max; break;  // own upper limit
    default: hit = false;
  }
  if (hit){ e.preventDefault(); setVal(i, n); }
});
