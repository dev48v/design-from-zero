📅 Date Picker — A Calendar Dropdown From Scratch

A text field that opens a real calendar.

Click the field below to drop open a month grid. The whole calendar is computed with plain Date math: the first cell is offset by the weekday the month starts on (new Date(y, m, 1).getDay()), then we fill in every day up to the month's length (new Date(y, m+1, 0).getDate() — which gives leap years for free). Use the ← / → arrows to change month (year rolls over automatically), the today cell is ringed, the selected day is filled. Click a day to drop it into the field and close. Bonus: arrow keys move the focused day, Enter selects, click-outside or Esc closes, and you can toggle disable past dates.

The month-grid math

A calendar is just a 7-column grid. Two Date calls build it. The first-day offset — new Date(year, month, 1).getDay() — tells you which weekday the 1st lands on (0 = Sunday), so you emit that many blank cells before day 1, which lines every date up under its weekday. The days-in-month — new Date(year, month + 1, 0).getDate() — asks for "day 0 of next month", which JavaScript reads as the last day of this month, so it returns 28/29/30/31 correctly, leap years included. Then you just loop 1…N emitting a cell per day. Navigation is month ± 1 on a Date and re-rendering; a value past 11 or below 0 rolls the year automatically.

The whole idea: a read-only input, a popover, and a 7-column grid. Offset the first cell by the month's starting weekday, fill in days-in-month, highlight today + the selected day, and changing month is just month ± 1 on a Date object — that is the entire date picker.

🔨 Build it yourself

    
A read-only input, a popover, and a 7-column grid built from two Date calls. Offset by the first weekday, fill days-in-month, then highlight today and the selection. Pure CSS plus vanilla JS, no libraries.

  3. 3
    render(year, month) — the two Date calls
    
        
    let view = new Date();                 // the month currently shown
view.setDate(1);                       // pin to the 1st (avoids overflow)

function render() {
  const y = view.getFullYear(), m = view.getMonth();   // m is 0..11!

  // which weekday does the 1st fall on?  0 = Sun
  const firstDow   = new Date(y, m, 1).getDay();
  // how many days in this month?  "day 0 of next month" = last of this
  const daysInMonth = new Date(y, m + 1, 0).getDate();  // leap years free

  dpTitle.textContent =
    view.toLocaleString("en", { month:"long", year:"numeric" });
  // ...build cells below
}
    
      
    4. 

      
  4. 4
    Offset with blanks, then fill the days
    
        
      let cells = "";
  // leading blanks so day 1 lands under its real weekday column
  for (let i = 0; i < firstDow; i++)
    cells += `<div class="dp-day blank"></div>`;

  // one cell per real day, 1 … daysInMonth
  for (let d = 1; d <= daysInMonth; d++) {
    cells += `<button class="dp-day" data-day="${d}">${d}</button>`;
  }
  dpDays.innerHTML = cells;

  // the Su..Sa header (once is fine, but easy here)
  dpDow.innerHTML = ["Su","Mo","Tu","We","Th","Fr","Sa"]
    .map(d => `<div class="dp-dow">${d}</div>`).join("");
    
      
    5. 

      
  5. 5
    Open / close + pick a day, format the field
    
        
    dpInput.onclick = () => { render(); dpPop.classList.remove("hidden"); };

dpDays.onclick = (e) => {
  const btn = e.target.closest(".dp-day[data-day]");
  if (!btn) return;
  const d = +btn.dataset.day;
  selected = new Date(view.getFullYear(), view.getMonth(), d);
  dpInput.value = selected.toLocaleDateString("en", {
    day:"numeric", month:"short", year:"numeric"   // e.g. 5 Jul 2026
  });
  dpPop.classList.add("hidden");      // close after picking
};

// click outside or Esc closes it too
document.addEventListener("click", (e) => {
  if (!dpWrap.contains(e.target)) dpPop.classList.add("hidden");
});

  6. 6
    Prev / next month (year rolls over for free)
    
        
    dpPrev.onclick = () => { view.setMonth(view.getMonth() - 1); render(); };
dpNext.onclick = () => { view.setMonth(view.getMonth() + 1); render(); };

// setMonth(-1) or setMonth(12) auto-adjusts the YEAR:
//   Jan 2026 → setMonth(-1) → Dec 2025
//   Dec 2026 → setMonth(12) → Jan 2027
// no manual year handling needed.
    
      
    7. 

      
  7. 7
    Highlight today + the selected day
    
        
    // compare by Y-M-D, NOT by timestamp (times would differ)
const sameDay = (a, b) => a && b &&
  a.getFullYear() === b.getFullYear() &&
  a.getMonth()    === b.getMonth() &&
  a.getDate()     === b.getDate();

// inside the day loop:
const cellDate = new Date(y, m, d);
let cls = "dp-day";
if (sameDay(cellDate, new Date())) cls += " today";     // ring it
if (sameDay(cellDate, selected))   cls += " selected";  // fill it
cells += `<button class="${cls}" data-day="${d}">${d}</button>`;
    
      
    8. 

      
  8. 8
    Keyboard nav + disable past dates
    
        
    // arrow keys move a "focused" day; Enter selects it
document.addEventListener("keydown", (e) => {
  if (dpPop.classList.contains("hidden")) return;
  const moves = { ArrowLeft:-1, ArrowRight:1, ArrowUp:-7, ArrowDown:7 };
  if (e.key in moves) {
    e.preventDefault();
    focusDate.setDate(focusDate.getDate() + moves[e.key]); // may cross months
    view = new Date(focusDate); view.setDate(1); render();
  }
  if (e.key === "Enter") selectDate(focusDate);
  if (e.key === "Escape") dpPop.classList.add("hidden");
});

// min = today: disable + block clicks on earlier days
const min = new Date(); min.setHours(0,0,0,0);
if (disablePast && cellDate < min) cls += " disabled";
