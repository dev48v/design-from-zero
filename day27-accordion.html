🪗 Accordion — Accessible Disclosure, Single vs Multi-Open, Animated Height + WAI-ARIA

Click a header, or focus one and press Enter · ↑ ↓ Home End.

This is a real accordion — a stack of expandable sections built from scratch in vanilla JS. Each header is a genuine <button> (not a clickable <div>), so it is focusable, in the tab order, and toggles for free on Enter / Space. Every button carries aria-expanded and aria-controls pointing at its panel; every panel is a role="region" labelled by its button. The open/close is a genuine height animation using the CSS grid-template-rows: 0fr → 1fr trick, so it slides even though the content height is unknown. Flip the toggle below to switch between single-open (opening one closes the rest) and allow-multiple. Focus a header and the ↑/↓ arrows move between headers, Home/End jump to first/last. It also respects prefers-reduced-motion.

Billing FAQ
Behaviour

Open panels: none. In single-open mode, expanding one section collapses whichever was open; in allow-multiple, they are independent.

Product details — allow-multiple, first open by default

A second accordion on the same page runs off the same factory with its own IDs. Panels hold arbitrary content — paragraphs, lists, links — and the grid-template-rows animation measures the height automatically, no scrollHeight math required.

Each header is a <button aria-expanded aria-controls> inside a heading; each panel is a role="region" with aria-labelledby pointing back at its button. This is the exact WAI-ARIA Accordion pattern used by FAQ pages, filter sidebars and settings screens across Stripe, GitHub and Notion.

🔨 Build it yourself

    
A heading + real <button> per section, a region panel wired with aria-expanded/aria-controls/aria-labelledby, the grid-template-rows: 0fr↔1fr height animation, a single-open vs allow-multiple state model, full keyboard navigation between headers, and a prefers-reduced-motion opt-out. ~70 lines of vanilla JS — no framework, no <details>, no dependencies.

    1
    Markup: heading > button, then a region panel
    
        
    Each item is a heading wrapping a real <button>, followed by the panel it controls. The button is focusable and toggles on Enter/Space for free — that is why it must be a button, not a <div>. aria-controls points at the panel's id; the panel is a role="region" labelled back by the button's id.
    
        
    <div class="acc-item">
  <h3 class="acc-header">
    <button class="acc-trigger" id="t0" aria-expanded="false" aria-controls="p0">
      Billing <svg class="acc-icon">…chevron…</svg>
    </button>
  </h3>
  <div class="acc-panel" id="p0" role="region" aria-labelledby="t0" data-open="false">
    <div class="acc-panel-inner"><div class="acc-panel-content">…</div></div>
  </div>
</div>

  2. 
        
    2
    CSS: the 0fr → 1fr grid height trick
    
        
    You cannot transition height:auto, so the panel is a one-row CSS grid whose track animates from 0fr (collapsed) to 1fr (its natural height). An inner wrapper with overflow:hidden clips the content while the row shrinks. Toggling one attribute drives the whole animation — no measuring, no inline styles.
    
        
    .acc-panel        { display:grid; grid-template-rows:0fr;
                    transition: grid-template-rows .3s ease; }
.acc-panel[data-open="true"] { grid-template-rows:1fr; }
.acc-panel-inner  { overflow:hidden; }   /* clip while the row collapses */

@media (prefers-reduced-motion: reduce){
  .acc-panel { transition:none; }        /* snap instead of slide */
}
    
      
    3. 
      
  3. 
        
    3
    Build the items and wire the ARIA
    
        
    Generate each item, cross-link the ids, and stash the buttons and panels in arrays so the rest of the code can index them. aria-expanded starts false; a data-open flag on the panel mirrors it and drives the CSS.
    
        
    items.forEach((item, i) => {
  btn.id = uid + "-t" + i;
  btn.setAttribute("aria-controls", uid + "-p" + i);
  btn.setAttribute("aria-expanded", "false");

  panel.id = uid + "-p" + i;
  panel.setAttribute("role", "region");
  panel.setAttribute("aria-labelledby", uid + "-t" + i);
  panel.setAttribute("data-open", "false");

  btn.addEventListener("click", () => toggle(i));
  btn.addEventListener("keydown", e => onKey(e, i));
  triggers.push(btn); panels.push(panel);
});
    
      
    4. 
      
  4. 
        
    4
    setOpen — one function keeps ARIA and CSS in sync
    
        
    There is a single source of truth: setOpen(i, open) writes aria-expanded on the button (for screen readers) and data-open on the panel (for the animation) together, so the accessible state and the visual state can never drift apart.
    
        
    function isOpen(i){ return panels[i].getAttribute("data-open") === "true"; }

function setOpen(i, open){
  triggers[i].setAttribute("aria-expanded", String(open));  // announced
  panels[i].setAttribute("data-open", String(open));        // animates
}
    
      
    5. 
      
  5. 
        
    5
    toggle — single-open vs allow-multiple
    
        
    Toggling flips the clicked item. The only difference between the two modes lives here: in single mode, before opening item i you close every other item; in multi mode each panel is independent. Clicking an open header always closes it, so an all-collapsed state is allowed.
    
        
    function toggle(i){
  const willOpen = !isOpen(i);
  if (mode === "single" && willOpen){
    for (let j = 0; j < n; j++) if (j !== i) setOpen(j, false);  // close siblings
  }
  setOpen(i, willOpen);
  live.textContent = items[i].title + (willOpen ? " expanded" : " collapsed");
}
    
      
    6. 
      
  6. 
        
    6
    Keyboard: move focus between headers
    
        
    Because the header is a button, Enter/Space already fire click → toggle, so you add nothing for activation. The accordion pattern adds navigation: ↑/↓ move focus to the previous/next header (wrapping), Home/End jump to the first/last. preventDefault() stops the arrows from scrolling the page.
    
        
    function onKey(e, i){
  let t = null;
  switch (e.key){
    case "ArrowDown": t = (i + 1) % n;       break;
    case "ArrowUp":   t = (i - 1 + n) % n;   break;
    case "Home":      t = 0;                 break;
    case "End":       t = n - 1;             break;
    default: return;                 // Enter/Space handled by the <button>
  }
  e.preventDefault();
  triggers[t].focus();
}

  7. 
        
    7
    Switch modes without losing state
    
        
    Flipping to single at runtime may leave several panels open, which now violates the rule — so collapse all but the first open one. Flipping to multi needs no cleanup. Wire the mode buttons to this and update their pressed styling.
    
        
    function setMode(m){
  mode = m;
  if (mode === "single"){
    let kept = false;
    for (let i = 0; i < n; i++)
      if (isOpen(i)){ if (kept) setOpen(i, false); else kept = true; }
  }
}
modeSingle.onclick = () => { acc.setMode("single"); paintModeButtons("single"); };
modeMulti .onclick = () => { acc.setMode("multi");  paintModeButtons("multi");  };

    8
    When to reach for an accordion (and when tabs)
    
        
    Use an accordion to declutter a long page of independent, mostly-scannable sections — FAQs, filters, settings groups — where users dip into one or two at a time and stay in a single vertical flow. Reach for tabs instead when the sections are peers and users switch between full views of the same size, showing exactly one at a time. Accordions can show none, one, or many panels; a tablist always shows exactly one panel — that is tomorrow's build.
