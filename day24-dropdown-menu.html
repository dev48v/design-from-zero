🍔 Dropdown / Menu Button — Roving Tabindex + Full Keyboard Nav

Click the button — or press Enter, Space or ↓ to open the menu.

This is a real WAI-ARIA menu button: a trigger that carries aria-haspopup and aria-expanded, and a role="menu" of role="menuitem" rows. Everything works from the keyboard. Open with Enter/Space/↓ (focus lands on the first item) or ↑ (focus lands on the last). Inside, only one item is tabbable at a time — roving tabindex — and ↑/↓ move with wrap-around, Home/End jump to the ends, and typeahead lets you type a letter to jump to a matching item. Enter/Space activate (you'll see a toast), Esc closes and returns focus to the button, and a click outside or a focus-out closes it too.

    Open: no
    Focused item: none
    Last action: —

    The button is aria-haspopup="true" + aria-expanded pointing at a role="menu" via aria-controls. Each row is a role="menuitem". Unlike the combobox from Day 23 (which keeps focus on the input and moves a virtual cursor with aria-activedescendant), a menu moves real DOM focus onto each item — the roving-tabindex model. Only the current item has tabindex="0"; every other item is tabindex="-1", so Tab leaves the menu rather than walking through it, and the arrow keys drive the highlight. This is the WAI-ARIA "menu button" pattern in ~120 lines of vanilla JS.

    🔨 Build it yourself
    
    
    A trigger button, a rendered role="menu", roving tabindex so only one item is tabbable, full arrow / Home / End / typeahead navigation, activate-and-close, Esc-and-outside-click dismissal that returns focus to the button, and the ARIA menu-button wiring. ~120 lines of vanilla JS — no framework, no dependencies.
    
    
      
      
    1. 
        
      1
      Markup: a button, not a <select>
      
        
      A real <button> plus a <ul role="menu"> you own. The button carries aria-haspopup="true" (a popup follows), aria-expanded (is it open?) and aria-controls pointing at the menu. Use a menu button — not a listbox or a native <select> — when the items are commands (Rename, Duplicate, Delete), not values you're picking from a set.
      
        
      <div class="mb-wrap">
  <button id="mbBtn" aria-haspopup="true" aria-expanded="false"
          aria-controls="mbMenu">Actions ▼</button>
  <ul id="mbMenu" role="menu" hidden></ul>
</div>
      
      
      2. 
      
    2. 
        
      2
      Render menuitems with roving tabindex
      
        
      Each row is a <li role="menuitem">. The trick is roving tabindex: exactly one item has tabindex="0" and every other has tabindex="-1". That's what makes Tab leave the menu (instead of stepping through every item) while the arrow keys still move focus internally. Skip disabled items when seeding the initial 0.
      
        
      menu.innerHTML = ITEMS.map((it, i) =>
  `<li role="menuitem" data-index="${i}" tabindex="-1"
       class="mb-item">${it.label}</li>`).join("");
// only the first (enabled) item is tabbable:
items()[0].tabIndex = 0;
      
      
      3. 
      
    3. 
        
      3
      Open — and which item gets focus
      
        
      Opening flips aria-expanded="true", unhides the menu, then moves real focus onto an item. The rule: a pointer click or Enter/Space/↓ focuses the first item; ↑ focuses the last. Passing a focusFirst flag keeps that one decision in one place.
      
        
      function open(focusFirst = true){
  menu.hidden = false;
  wrap.classList.add("open");
  btn.setAttribute("aria-expanded", "true");
  const list = items();
  focusItem(focusFirst ? 0 : list.length - 1);
}
      
      
      4. 
      
    4. 
        
      4
      focusItem() — the roving move
      
        
      Moving the highlight is one function. Set the previous item back to tabindex="-1", give the new one tabindex="0", then call .focus() on it. Because focus itself moves, screen readers announce the item automatically — no aria-activedescendant needed. Skip disabled items so focus never lands on a dead command.
      
        
      function focusItem(i){
  const list = items();
  i = (i + list.length) % list.length;      // wrap
  while (list[i].getAttribute("aria-disabled") === "true")
    i = (i + 1) % list.length;              // skip disabled
  list.forEach(el => el.tabIndex = -1);
  list[i].tabIndex = 0;
  list[i].focus();
  cur = i;
}
      
      
      5. 
      
    5. 
        
      5
      Arrow / Home / End navigation
      
        
      The keydown lives on the menu. ↓ is cur+1, ↑ is cur-1 — focusItem() already wraps with modulo. Home jumps to the first item, End to the last. Call preventDefault() so arrows don't scroll the page.
      
        
      menu.addEventListener("keydown", e => {
  if (e.key === "ArrowDown"){ e.preventDefault(); focusItem(cur + 1); }
  else if (e.key === "ArrowUp"){ e.preventDefault(); focusItem(cur - 1); }
  else if (e.key === "Home"){ e.preventDefault(); focusItem(0); }
  else if (e.key === "End"){ e.preventDefault(); focusItem(items().length - 1); }
});

    6. 
        
      6
      Typeahead — type a letter to jump
      
        
      A single printable key jumps to the next item whose label starts with that letter. Keep a small buffer that resets after ~500ms so typing "de" quickly matches "Delete" while a slow second key starts a fresh search. Search from the item after the current one and wrap, so repeated presses of the same letter cycle through matches.
      
        
      let buf = "", bufT;
function typeahead(ch){
  clearTimeout(bufT); bufT = setTimeout(() => buf = "", 500);
  buf += ch.toLowerCase();
  const list = items();
  for (let k = 1; k <= list.length; k++){
    const j = (cur + k) % list.length;
    if (list[j].dataset.label.startsWith(buf)){ focusItem(j); return; }
  }
}
      
      
      7. 
      
    7. 
        
      7
      Activate — Enter, Space, click
      
        
      Enter and Space activate the focused item; a click activates the clicked one. Activating a menuitem runs its command and closes the menu, then returns focus to the button so the next Tab continues from where the user was. One activate(i) function serves mouse and keyboard so behaviour is identical.
      
        
      if (e.key === "Enter" || e.key === " "){ e.preventDefault(); activate(cur); }
menu.addEventListener("click", e => {
  const li = e.target.closest("[data-index]"); if (li) activate(+li.dataset.index);
});
function activate(i){
  if (items()[i].getAttribute("aria-disabled") === "true") return;
  run(ITEMS[+items()[i].dataset.index].label);   // do the thing
  close(true);                                    // then close + return focus
}
      
      
      8. 
      
    8. 
        
      8
      Dismiss — Esc, outside click, focus-out
      
        
      Esc closes and returns focus to the button. An outside pointerdown closes without moving focus. A focusout that lands outside the widget (Tab, or clicking another control) closes it too. close(returnFocus) hides the menu, flips aria-expanded="false", and only re-focuses the button when the dismissal was keyboard-driven.
      
        
      if (e.key === "Escape"){ e.preventDefault(); close(true); }
document.addEventListener("pointerdown", e => {
  if (!wrap.contains(e.target)) close(false);
});
wrap.addEventListener("focusout", e => {
  if (!wrap.contains(e.relatedTarget)) close(false);
});
function close(returnFocus){
  menu.hidden = true; wrap.classList.remove("open");
  btn.setAttribute("aria-expanded", "false");
  if (returnFocus) btn.focus();
}

      9
      Submenus (bonus)
      
        
      A menuitem that opens a submenu is a role="menuitem" with aria-haspopup="true" and its own nested role="menu". → (or Enter) opens it and focuses its first item; ← (or Esc) closes it and returns focus to the parent item. Each menu runs the same roving-tabindex + arrow logic — it's this widget, recursively nested.
      
        
      <li role="menuitem" aria-haspopup="true" aria-expanded="false">
  Export…
  <ul role="menu" hidden>
    <li role="menuitem" tabindex="-1">PDF</li>
    <li role="menuitem" tabindex="-1">CSV</li>
  </ul>
</li>
// ArrowRight → open submenu, focus its first item
// ArrowLeft / Esc → close submenu, focus parent item
