🪟 Modal Dialog — Focus Trap, Esc, and Done Right

A dialog that traps focus and locks the page behind it.

Click a button below to open a real modal: it fades the backdrop in, scales the card up, moves focus inside, and freezes the page behind it. Try every way out — the × button, a click on the dimmed backdrop, or the Esc key. While it is open, press Tab and Shift+Tab: focus cycles inside the dialog and can never reach the page underneath. Notice the page does not scroll while it is open, and when you close, focus returns to the exact button you opened it with — no getting stranded. All vanilla JS and CSS, with role="dialog", aria-modal="true", and a labelled title.

Why a modal is more than a centered box

A modal interrupts the user on purpose: it puts one decision in front of everything else and dims the rest. That promise only holds if the dialog actually captures the user — focus must stay inside it (a focus trap), the page behind must not scroll, every obvious exit must work, and focus must come back to where it started. Skip any of these and a keyboard or screen-reader user can Tab straight off into the hidden page behind the dim, click controls they can't see, and lose their place entirely.

What's new ℹ︎
This dialog traps focus, locks page scroll, closes on Esc or a backdrop click, and returns focus to the button that opened it. That is the whole accessibility contract for a modal — and it is only a screenful of code.
Hand-rolled vs the native <dialog> element

Everything here is built by hand so you can see each piece. The browser also ships a real element — <dialog> with .showModal() — that gives you the focus trap, the Esc-to-close, the backdrop (::backdrop), aria-modal, and inertness of the page behind it for free. In production you'd usually reach for that. Build it once by hand, though, and you'll understand exactly what showModal() is doing under the hood — and why a centered <div> alone is not a modal.

The whole idea: a real modal isn't just a centered box — it traps focus, locks scroll, closes on Esc/backdrop, and returns focus, or it's an accessibility trap.

🔨 Build it yourself

    
A trigger button, a fixed overlay, and a centered card. Fade the backdrop, scale the card, then add the four things that make it a real modal: Esc/backdrop close, a focus trap, a scroll lock, and return-focus. Pure CSS plus vanilla JS, no libraries — then the one-line native alternative.

      <button class="md-btn primary"   data-close>Confirm</button>
    </div>
  </div>
</div>

  2. 2
    Style: full-screen overlay, centered card, fade + scale
    
        
    /* overlay covers the viewport and centers its child */
.md-overlay {
  position:fixed; inset:0; z-index:60;
  background:rgba(15,23,42,.55);          /* the dim */
  display:flex; align-items:center; justify-content:center;
  opacity:0; transition:opacity .2s ease;
}
.md-overlay.open   { opacity:1; }         /* fade the backdrop in */
.md-overlay.hidden { display:none; }      /* fully removed when closed */

/* the card starts a touch small + transparent, then scales in */
.md-dialog {
  background:#fff; border-radius:18px;
  transform:scale(.94); opacity:0;
  transition:transform .2s ease, opacity .2s ease;
}
.md-overlay.open .md-dialog { transform:scale(1); opacity:1; }
    
      
    3. 

      
  3. 3
    open() / close() — flip hidden, then animate
    
        
    function open(overlay) {
  overlay.classList.remove("hidden");     // mount it (display:flex)
  // next frame: add .open so the transition actually runs
  requestAnimationFrame(() => overlay.classList.add("open"));
}

function close(overlay) {
  overlay.classList.remove("open");       // play fade/scale OUT
  overlay.addEventListener("transitionend", function done() {
    overlay.classList.add("hidden");      // unmount AFTER the animation
    overlay.removeEventListener("transitionend", done);
  }, { once:true });
}

// wire the triggers
document.querySelectorAll("[data-open]").forEach(btn =>
  btn.onclick = () => open(document.getElementById(btn.dataset.open)));
    
      
    4. 

      
  4. 4
    Close on ×, on backdrop click, and on Esc — all three
    
        
    // × buttons and any [data-close] inside the dialog
overlay.querySelectorAll("[data-close]").forEach(el =>
  el.onclick = () => close(overlay));

// backdrop click: only when the click is the OVERLAY itself,
// not a click that bubbled up from inside the card
overlay.addEventListener("mousedown", (e) => {
  if (e.target === overlay) close(overlay);
});

// Esc closes the topmost open dialog
document.addEventListener("keydown", (e) => {
  if (e.key === "Escape" && current) close(current);
});
    
      
    5. 

      
  5. 5
    The focus trap — keep Tab inside the dialog
    
        
    const SEL = 'a[href], button:not([disabled]), input, textarea, ' +
            'select, [tabindex]:not([tabindex="-1"])';

function trap(e, dialog) {
  if (e.key !== "Tab") return;
  const f = [...dialog.querySelectorAll(SEL)]
              .filter(el => el.offsetParent !== null);   // visible only
  if (!f.length) return;
  const first = f[0], last = f[f.length - 1];

  if (e.shiftKey && document.activeElement === first) {
    e.preventDefault(); last.focus();   // wrap to the end
  } else if (!e.shiftKey && document.activeElement === last) {
    e.preventDefault(); first.focus();  // wrap to the start
  }
}
// on open, move focus INTO the dialog (first field/button)
dialog.querySelector(SEL)?.focus();
    
      
    6. 

      
  6. 6
    Lock body scroll (and pad for the scrollbar)
    
        
    function lockScroll() {
  // the scrollbar vanishes when overflow is hidden — its width would
  // make the page JUMP. Pad the body by that exact width to hold still.
  const sbw = window.innerWidth - document.documentElement.clientWidth;
  document.body.style.paddingRight = sbw + "px";
  document.documentElement.classList.add("md-scroll-locked");  // overflow:hidden
}
function unlockScroll() {
  document.documentElement.classList.remove("md-scroll-locked");
  document.body.style.paddingRight = "";
}
/* .md-scroll-locked { overflow:hidden; } */
    
      
    7. 

      
  7. 7
    Return focus to the trigger on close
    
        
    let lastTrigger = null;

function open(overlay) {
  lastTrigger = document.activeElement;   // remember who opened us
  overlay.classList.remove("hidden");
  requestAnimationFrame(() => overlay.classList.add("open"));
  lockScroll();
  overlay.querySelector(SEL)?.focus();    // focus into the dialog
}

function close(overlay) {
  overlay.classList.remove("open");
  unlockScroll();
  lastTrigger?.focus();   // DON'T strand the user — send focus back
  // …then unmount on transitionend as in step 3
}

  <p>…content…</p>
  <form method="dialog">            <!-- a submit auto-closes it -->

    <button value="ok">Confirm</button>
  </form>
</dialog>
