🚪 Drawer — A Slide-in Panel From Any Edge, With a Scrim, Focus Trap, Scroll-Lock, Swipe-to-Close & Dialog A11y

Slide a real panel in from any edge.

A working drawer (off-canvas panel) built from scratch in vanilla JS. Pick an edge and hit Open: the panel slides in on a transform transition while a scrim fades in behind it. Focus jumps into the drawer and is trapped — Tab cycles through its controls and wraps around instead of leaking to the page behind. Close it with Esc, by clicking the scrim, with the × button, or by dragging the grabber toward the edge past a threshold. Focus returns to the button that opened it, and the page scroll stays locked the whole time.

Edge
State closed right focus-trap scroll-lock Tab to cycle · Esc / scrim to close · drag the grabber to dismiss

This is the panel behind every mobile menu, cart, filter sheet and settings pane. Under the hood it is one fixed element parked off-screen with a transform, a backdrop that fades in, and a small amount of focus + scroll bookkeeping so it behaves like a real modal dialog — no library.

🔨 Build it yourself

    
A drawer is one fixed panel parked off the edge with a transform, slid in on a transition with a fading scrim behind it, plus the focus + scroll bookkeeping that makes it behave like a real modal dialog: focus moves in and is trapped, the background scroll is locked, Esc and the scrim close it, focus returns to the trigger, and a pointer drag dismisses it. ~110 lines of vanilla JS, no dependencies.

    
    
      
  1. 
        
    1
    Park the panel off-canvas
    
        
    A drawer is a position:fixed element pinned to one edge and pushed just out of view with a transform. A left drawer sits at left:0 translated -100% on X; a bottom sheet sits at bottom:0 translated 100% on Y. It is fully laid out and ready — simply parked past the edge until you need it.
    
        
    .drawer { position:fixed; z-index:61; background:#fff; }

.drawer.left   { top:0; left:0;  height:100%; width:360px;  transform:translateX(-100%); }
.drawer.right  { top:0; right:0; height:100%; width:360px;  transform:translateX(100%);  }
.drawer.top    { top:0; left:0;  width:100%;  height:320px; transform:translateY(-100%); }
.drawer.bottom { bottom:0; left:0; width:100%; height:320px; transform:translateY(100%); }
    
      
    2. 
      
  2. 
        
    2
    Slide it in with transform + transition
    
        
    Opening is one class. .open sets transform:none and a transition animates from the parked offset to zero. Animate transform, not left/display: transform is GPU-composited (cheap, 60fps) and — unlike display:none — it is actually animatable, so you get a real slide instead of an instant pop.
    
        
    .drawer      { transition: transform .32s cubic-bezier(.4,0,.2,1); will-change: transform; }
.drawer.open { transform: none; }        /* slide to its resting position */

// open / close is just a class toggle:
drawer.classList.add("open");            // slides in
drawer.classList.remove("open");         // slides back out
    
      
    3. 
      
  3. 
        
    3
    The scrim — fade a backdrop, click to close
    
        
    A full-screen backdrop below the drawer does two jobs: it dims the page so the panel reads as "modal", and it is the click-outside target. Fade it with opacity (also toggle visibility so it can't be clicked while transparent), and wire a click on it straight to close.
    
        
    .scrim      { position:fixed; inset:0; background:rgba(15,23,42,.5);
              opacity:0; visibility:hidden; transition:opacity .3s, visibility .3s; z-index:60; }
.scrim.open { opacity:1; visibility:visible; }

scrim.addEventListener("click", closeDrawer);   // click-outside to dismiss
    
      
    4. 
      
  4. 
        
    4
    Move focus in — and remember the trigger
    
        
    When the drawer opens, keyboard focus must follow. Remember the element that opened it (so you can restore focus later), then move focus to the first focusable control inside the panel. This is what lets a keyboard or screen-reader user actually operate the thing they just opened.
    
        
    const FOCUSABLE = "a[href], button:not([disabled]), input:not([disabled])," +
                  "select:not([disabled]), textarea, [tabindex]:not([tabindex='-1'])";

function openDrawer(trigger){
  lastTrigger = trigger || document.activeElement;   // remember who opened us
  drawer.classList.add("open"); scrim.classList.add("open");
  const f = drawer.querySelectorAll(FOCUSABLE);
  (f[0] || drawer).focus();                           // move focus into the panel
}
    
      
    5. 
      
  5. 
        
    5
    Trap focus — wrap first ↔ last
    
        
    Focus must not escape a modal drawer to the page behind it. Collect the focusable elements; when Tab is pressed on the last one, send focus back to the first; when Shift+Tab is pressed on the first, send it to the last. That two-line wrap is the whole focus trap.
    
        
    function onKeydown(e){
  if (e.key !== "Tab") return;
  const f = focusables();
  const first = f[0], last = f[f.length - 1];
  if (e.shiftKey && document.activeElement === first){ e.preventDefault(); last.focus(); }
  else if (!e.shiftKey && document.activeElement === last){ e.preventDefault(); first.focus(); }
}
    
      
    6. 
      
  6. 
        
    6
    Lock the background scroll
    
        
    While the drawer is open the page behind it must not scroll. Set overflow:hidden on <body>. But removing the scrollbar makes the page jump wider by its width — so measure the scrollbar (innerWidth − clientWidth) and add it back as padding-right. On close, restore both.
    
        
    function lockScroll(){
  const sbw = window.innerWidth - document.documentElement.clientWidth;  // scrollbar width
  document.body.style.overflow = "hidden";
  if (sbw > 0) document.body.style.paddingRight = sbw + "px";            // no layout jump
}
function unlockScroll(){
  document.body.style.overflow = "";
  document.body.style.paddingRight = "";
}
    
      
    7. 
      
  7. 
        
    7
    Close paths + return focus
    
        
    Every dismissal funnels through one closeDrawer(): Esc, the scrim, the × button, the swipe. It removes .open, unlocks scroll, and — critically — returns focus to the element you saved when opening, so the keyboard user lands exactly where they left off instead of at the top of the page.
    
        
    function closeDrawer(){
  drawer.classList.remove("open"); scrim.classList.remove("open");
  unlockScroll();
  document.removeEventListener("keydown", onKeydown, true);
  if (lastTrigger) lastTrigger.focus();     // restore focus to the opener
}
document.addEventListener("keydown", e => { if (e.key === "Escape") closeDrawer(); });
    
      
    8. 
      
  8. 
        
    8
    Dialog a11y, swipe-to-close & reduced motion
    
        
    Give the panel role="dialog" aria-modal="true" and label it with aria-labelledby; mark it inert while closed so nothing inside is tabbable. For swipe, follow the pointer with an inline transform (transition off), and on release close if the drag passed a third of the panel — else snap back. Finally, honour prefers-reduced-motion by dropping the transitions.
    
        
    <aside role="dialog" aria-modal="true" aria-labelledby="t" aria-hidden="true" inert>…</aside>

function onDrag(e){                                   // follow the finger, transition off
  const off = edge === "right" ? Math.max(0, e.clientX - startX) : 0; /* per edge */
  drawer.style.transform = "translateX(" + off + "px)";
}
function endDrag(e){
  if (Math.abs(e.clientX - startX) > size * 0.33) closeDrawer();  // past threshold → close
  else drawer.style.transform = "";                               // else snap back
}
@media (prefers-reduced-motion: reduce){ .drawer, .scrim { transition:none; } }
