🎚 Segmented Control — Sliding Thumb + Radiogroup Keyboard Nav

Click a segment — or focus one and press ← → Home End.

This is an iOS-style segmented control: two-to-five mutually-exclusive options laid out in a single pill, with a white thumb that slides under the selected label. The thumb is animated with a CSS transform — its position and width are measured from each segment, so it lands perfectly even when the segments are different widths. We give it role="radiogroup" (with role="radio" + aria-checked children), not a tablist, because these options set a value that persists — they don't switch between visible views. Only one segment is tabbable at a time (roving tabindex), Tab treats the whole group as one stop, and inside it the arrow keys move focus and change the selection together with wrap-around, while Home/End jump to the ends.

The group is role="radiogroup"; each option is role="radio" with aria-checked. Only the checked option has tabindex="0" — every other is tabindex="-1" — so Tab enters and leaves the control as a unit and the arrows drive selection inside it. This is the same composite-widget model behind iOS's UISegmentedControl, SwiftUI's Picker(.segmented), and the "grid / list" toggles in GitHub, Linear and Figma.

🔨 Build it yourself

    
A pill track, a set of role="radio" buttons, a white thumb whose size and position are measured from the chosen segment and animated with transform, roving tabindex so only one option is tabbable, arrow-key selection with wrap-around plus Home/End, click-to-select, a resize re-measure, a no-flash first paint, and a reduced-motion opt-out. ~70 lines of vanilla JS — no framework, no dependencies.

    
    
      
  1. 
        
    1
    Markup: a radiogroup + a thumb
    
        
    The container is role="radiogroup" with an accessible label; each option is a <button role="radio"> carrying aria-checked. A single empty <div class="seg-thumb"> is the sliding pill — it lives inside the track so its offsets share the track's coordinate system. Use a radiogroup (not a tablist) because these options set a value that sticks, they don't reveal panels.
    
        
    <div class="seg" role="radiogroup" aria-label="View">
  <div class="seg-thumb"></div>
  <button role="radio" aria-checked="true"  tabindex="0">List</button>
  <button role="radio" aria-checked="false" tabindex="-1">Grid</button>
  <button role="radio" aria-checked="false" tabindex="-1">Board</button>
</div>
    
      
    2. 
      
  2. 
        
    2
    CSS: track, thumb, and layering
    
        
    The track is position:relative so the absolutely-positioned thumb is measured against it. Give the thumb z-index:1 and the labels z-index:2 so text sits on top of the pill. The checked label just changes colour; the moving pill is the whole visual. Animate only transform (and, if segments differ, width) so the browser can composite the slide.
    
        
    .seg      { position:relative; display:inline-flex; padding:4px;
            background:#eef2ff; border-radius:14px; }
.seg-thumb{ position:absolute; top:0; left:0; z-index:1; border-radius:10px;
            background:#fff; box-shadow:0 2px 8px rgba(15,23,42,.14);
            transition:transform .28s cubic-bezier(.34,1.2,.44,1),
                       width .28s cubic-bezier(.34,1.2,.44,1); }
.seg-opt  { position:relative; z-index:2; border:none; background:none;
            cursor:pointer; color:#475569; }
.seg-opt[aria-checked="true"]{ color:#4f46e5; }
    
      
    3. 
      
  3. 
        
    3
    Seed roving tabindex
    
        
    Exactly one option is tabbable: the checked one gets tabindex="0", all others -1. That's what makes Tab land on the group once and then leave — the arrows handle movement inside. Grab the option list up front; a live querySelectorAll also works if the segments can change.
    
        
    const opts = [...track.querySelectorAll('[role="radio"]')];
let cur = opts.findIndex(o => o.getAttribute("aria-checked") === "true");
if (cur < 0) cur = 0;
opts.forEach((o, i) => o.tabIndex = i === cur ? 0 : -1);
    
      
    4. 
      
  4. 
        
    4
    Measure & move the thumb
    
        
    This is the heart of it. Read the chosen segment's offsetLeft, offsetTop, offsetWidth and offsetHeight — all relative to the track, since the thumb starts at 0,0 inside it — then set the thumb's width/height and translate to match. Because we measure rather than compute i / count, this is exactly what makes unequal-width segments work.
    
        
    function moveThumb(){
  const seg = opts[cur];
  thumb.style.width  = seg.offsetWidth  + "px";
  thumb.style.height = seg.offsetHeight + "px";
  thumb.style.transform =
    `translate(${seg.offsetLeft}px, ${seg.offsetTop}px)`;
}
    
      
    5. 
      
  5. 
        
    5
    select(i) — the single state change
    
        
    One function does everything: wrap the index, flip aria-checked, rove the tabindex, move the thumb, optionally move focus, and fire an onChange. Mouse and keyboard both call it, so behaviour is identical. Wrapping with ((i % n) + n) % n handles the negative index you get pressing ← on the first item.
    
        
    function select(i, focus){
  const n = opts.length;
  i = ((i % n) + n) % n;                 // wrap both ends
  opts.forEach((o, j) => {
    o.setAttribute("aria-checked", j === i ? "true" : "false");
    o.tabIndex = j === i ? 0 : -1;       // roving tabindex
  });
  cur = i;
  moveThumb();
  if (focus) opts[i].focus();
  onChange && onChange(opts[i], i);
}
    
      
    6. 
      
  6. 
        
    6
    Keyboard: arrows select, Home/End jump
    
        
    In a radiogroup the arrows move focus and change selection at the same time — there's no "focus without checking" step. Right/Down go cur+1, Left/Up go cur-1 (both wrap), Home is index 0, End the last. Call preventDefault() so the arrows don't scroll the page.
    
        
    track.addEventListener("keydown", e => {
  const k = e.key;
  if (k === "ArrowRight" || k === "ArrowDown"){ e.preventDefault(); select(cur + 1, true); }
  else if (k === "ArrowLeft" || k === "ArrowUp"){ e.preventDefault(); select(cur - 1, true); }
  else if (k === "Home"){ e.preventDefault(); select(0, true); }
  else if (k === "End"){ e.preventDefault(); select(opts.length - 1, true); }
});

  7. 
        
    7
    Click, resize, and a no-flash first paint
    
        
    A click selects the clicked option. On resize the segment widths change, so re-run moveThumb() (guard against a hidden container where offsets read 0). For the very first paint, disable the transition, place the thumb, then restore it — otherwise the pill visibly slides in from the top-left corner on load.
    
        
    track.addEventListener("click", e => {
  const o = e.target.closest('[role="radio"]');
  if (o) select(opts.indexOf(o), true);
});
addEventListener("resize", () => { if (track.offsetParent) moveThumb(); });
// no-flash: position once with the animation off
thumb.style.transition = "none";
moveThumb();
requestAnimationFrame(() => { thumb.style.transition = ""; });

  8. 
        
    8
    Respect reduced motion — and know when to use it
    
        
    Some people get motion sick from sliding UI, so honour their OS setting: one media query kills the thumb transition while keeping the instant jump to the new position. Reach for a segmented control when you have 2–5 short, mutually-exclusive options that apply instantly and don't need to be scrolled — a view switch, a units toggle, a filter. Beyond five options, or when the choices are commands or need scrolling, use a select, a menu, or tabs instead.
    
        
    @media (prefers-reduced-motion: reduce){
  .seg-thumb { transition: none; }   /* jump, don't slide */
}
/* Use it for: List / Grid / Board · Day / Week / Month · °C / °F
   Not for: 8+ options, commands, or anything that scrolls */
