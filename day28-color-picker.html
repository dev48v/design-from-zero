🎨 Color Picker — HSV Saturation-Value Square + Hue/Alpha Sliders, Live HEX/RGB/HSL & Two-Way Binding

Drag inside the square, slide the hue & alpha rails, or type a value.

This is a real color picker built from scratch in vanilla JS — no <input type="color">. Dragging the big saturation / value square sets how vivid and how bright the color is; the vertical hue rail spins the base color through 0–360°; the alpha rail controls opacity over a checkerboard. Every format below stays in sync — HEX, RGB, RGBA, HSL — and the binding is two-way: type #10b981 into the HEX box and the square, rails and preview all jump to it. All the conversions (HSV↔RGB↔HEX↔HSL) are hand-written. It even reports the WCAG contrast ratio against white and black, and offers the native EyeDropper where the browser supports it.

🔨 Build it yourself

    
A saturation/value square drawn with two stacked CSS gradients, a hue rail and a checkerboard alpha rail, hand-written HSV↔RGB↔HEX↔HSL conversions, a pointer-drag helper that maps cursor position to sat/val, one render() that is the single source of truth, two-way text inputs that parse a value back into the picker, plus a WCAG contrast readout and the native EyeDropper. ~120 lines of vanilla JS — no framework, no <input type="color">, no dependencies.

    Markup: the SV square + two rails
    
        
    Three elements: the saturation/value square, the hue rail, and the alpha rail (over a checkerboard). Each holds a thumb the code positions absolutely. No native inputs anywhere.

<div class="cp-hue"   id="hue"><div class="cp-slider-thumb" id="hueThumb"></div></div>
<div class="cp-alpha checker" id="alpha">
  <div class="cp-alpha-fill" id="alphaFill"></div>
  <div class="cp-slider-thumb" id="alphaThumb"></div>
</div>

  2. 
        
    2
    The SV square is two stacked gradients
    
        
    The base background-color is the pure hue (hsl(h,100%,50%)), set from JS. On top: a white→transparent gradient left-to-right is saturation, and a transparent→black gradient top-to-bottom is value. Top-left = white, top-right = full hue, bottom = black. The thumb sits at (sat·W, (1−val)·H).
    
        
    .cp-sv {
  background-color: hsl(0,100%,50%);          /* JS updates the hue */
  background-image:
    linear-gradient(to top,   #000, transparent),   /* value  ↓ black */
    linear-gradient(to right, #fff, transparent);   /* satur. ← white */
}
/* thumb: left = sat*100%, top = (1-val)*100% */

  3. 
        
    3
    Hue rainbow + checkerboard alpha rail
    
        
    The hue rail is a single 7-stop rainbow so top and bottom are both red (0° = 360°). The alpha rail is a checkerboard (so transparency is visible) with a color→transparent gradient painted over it from JS.
    
        
    .cp-hue { background: linear-gradient(to bottom,
  #f00 0%, #ff0 17%, #0f0 33%, #0ff 50%, #00f 67%, #f0f 83%, #f00 100%); }

.checker { background-image:                    /* 4 gradients = squares */
  linear-gradient(45deg,#cbd5e1 25%,transparent 25%), ... ;
  background-size:14px 14px; }
/* alphaFill (JS): linear-gradient(to bottom, rgb(r,g,b), transparent) */
    
      
    4. 
      
  4. 
        
    4
    The conversions: HSV → RGB → HEX
    
        
    HSV is what the UI stores; RGB is what CSS wants; HEX is what people paste. Write them by hand — it is a few lines each and the round-trip is exact.
    
        
    function hsvToRgb(h,s,v){
  const c=v*s, x=c*(1-Math.abs((h/60)%2-1)), m=v-c;
  let [r,g,b]=[[c,x,0],[x,c,0],[0,c,x],[0,x,c],[x,0,c],[c,0,x]][Math.floor(h/60)%6];
  return { r:Math.round((r+m)*255), g:Math.round((g+m)*255), b:Math.round((b+m)*255) };
}
const hex2 = n => n.toString(16).padStart(2,"0");
const rgbToHex = (r,g,b) => "#"+hex2(r)+hex2(g)+hex2(b);
    
      
    5. 
      
  5. 
        
    5
    Pointer drag → sat / val
    
        
    One tiny helper turns any element into a draggable pad: on pointerdown it captures the pointer and reports the normalized 0–1 position on every move. setPointerCapture keeps the drag alive even when the cursor leaves the box.
    
        
    function drag(el, on){
  el.addEventListener("pointerdown", e => {
    el.setPointerCapture(e.pointerId);
    const move = ev => { const r = el.getBoundingClientRect();
      on(clamp((ev.clientX-r.left)/r.width,0,1),
         clamp((ev.clientY-r.top)/r.height,0,1)); };
    move(e); el.onpointermove = move;
    el.onpointerup = () => el.onpointermove = null;
  });
}
drag(sv,  (x,y) => { sat=x; val=1-y; render(); });  // square
drag(hue, (x,y) => { hue=y*360;      render(); });  // rail (y only)
    
      
    6. 
      
  6. 
        
    6
    render() — one source of truth
    
        
    hue/sat/val/alpha is the only state. render() derives everything from it — the square's hue, every thumb position, the alpha gradient, the preview and all four text fields — so nothing can drift. Skip the field the user is currently typing in so you never yank their cursor.
    
        
    function render(){
  const rgb = hsvToRgb(hue,sat,val), hex = rgbToHex(rgb.r,rgb.g,rgb.b);
  sv.style.backgroundColor = "hsl("+Math.round(hue)+",100%,50%)";
  svThumb.style.left = sat*100+"%";  svThumb.style.top = (1-val)*100+"%";
  hueThumb.style.top = hue/360*100+"%"; alphaThumb.style.top = (1-alpha)*100+"%";
  set(hexIn, hex); set(rgbIn, "rgb("+rgb.r+", "+rgb.g+", "+rgb.b+")"); /* ...hsl */
}
const set = (el,v) => { if (document.activeElement !== el) el.value = v; };

  7. 
        
    7
    Two-way binding: parse a value back in
    
        
    The inputs are not read-only. On every keystroke, parse the text; if it is a valid color, convert it back to HSV, overwrite the state and render(). Now typing #10b981 moves the square, both rails and the preview.
    
        
    hexIn.addEventListener("input", () => {
  const rgb = hexToRgb(hexIn.value.trim());
  if (!rgb) return;                       // ignore half-typed junk
  const c = rgbToHsv(rgb.r, rgb.g, rgb.b);
  if (c.s !== 0) hue = c.h;                // keep hue on greys
  sat = c.s; val = c.v; render();
});

    Contrast ratio + native EyeDropper
    
        
    Compute WCAG relative luminance and the contrast ratio against white/black so a picked color can be checked for legibility (AA needs ≥ 4.5:1). And if the browser exposes window.EyeDropper, one click samples any pixel on screen.
    
        
    const lum = c => { const f = x => { x/=255;
  return x<=.03928 ? x/12.92 : ((x+.055)/1.055)**2.4; };
  return .2126*f(c.r)+.7152*f(c.g)+.0722*f(c.b); };
function contrast(a,b){ const [hi,lo]=[lum(a),lum(b)].sort((p,q)=>q-p);
  return (hi+.05)/(lo+.05); }              // 1 (none) .. 21 (black/white)

if (window.EyeDropper)                     // Chromium only, feature-detect
  new EyeDropper().open().then(r => setHex(r.sRGBHex));
