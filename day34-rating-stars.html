⭐ Rating Stars — Hover-Preview vs Committed Value, Half-Star Precision, A Slider-Style Keyboard Model & Full ARIA

Point, hover, click — tell us how it was.

A working star-rating input built from scratch in vanilla JS. The trick behind every rating widget is two values, not one: the committed rating you have actually chosen, and a throwaway hover preview that paints the stars as your pointer sweeps across them without changing anything until you click. Turn on half-stars and each star splits down the middle — the left half is x.5, the right half is x. It is fully keyboard-driven too: Tab to the stars, then ← → to adjust, Home/End for min/max. No library.

Hover to preview, click to set — or focus the stars and use the arrow keys.

🔨 Build it yourself

    
A star rating is deceptively small: a MAX, a committed value, a hover preview that is null when idle, and a step (1 or 0.5). The stars are drawn as two stacked layers so any fraction can be shown by clipping the gold layer's width. Hover, click and the keyboard all do the same thing — change one of those two numbers and re-render. ~90 lines of vanilla JS, no dependencies.

    
    
      
  1. 
        
    1
    Two numbers: committed value + hover preview
    
        
    The whole component turns on a single idea. Keep the rating the user has actually chosen in value, and keep a separate hover that is null unless the pointer (or a key) is actively previewing. What the stars display is whichever one is in charge right now — hover if it is set, otherwise value. Hovering never touches value; only a click or a keypress commits.
    
        
    const MAX = 5;
let value = 0;        // committed rating — survives after the pointer leaves
let hover = null;     // transient preview; null means "not previewing"
let step  = 0.5;      // 0.5 = half-stars, 1 = whole stars only

// what the stars should paint RIGHT NOW:
const displayValue = () => hover !== null ? hover : value;
    
      
    2. 
      
  2. 
        
    2
    Draw each star as two stacked layers
    
        
    You cannot paint "half a star" with a single glyph, so stack two identical stars: a grey empty layer underneath and a gold fill layer on top. The fill layer is a clipping window — set its width to a percentage and it reveals only that slice of the gold star. The inner SVG keeps its full pixel width so it does not squash as the window narrows. This is what lets one mechanism show 0%, 50%, 100% — or any fraction.
    
        
    <span class="star">
  <span class="star-bg"><svg>…star…</svg></span>   <!-- grey, full -->
  <span class="star-fill"><svg>…star…</svg></span> <!-- gold, clipped -->
</span>

.star      { position:relative; width:46px; height:46px; }
.star-fill { position:absolute; inset:0 auto 0 0; width:0; overflow:hidden; }
.star-fill svg { width:46px; }   /* fixed → the window clips, glyph never shrinks */
    
      
    3. 
      
  3. 
        
    3
    Render: turn one number into N fill widths
    
        
    Rendering is a pure projection of displayValue(). For star i (1-based) the portion filled is displayValue − (i−1), clamped to the 0–1 range: star 1 fills first, then star 2, and so on. Multiply by 100 for the CSS width. Re-run this after every change — never nudge one star by hand.
    
        
    function render(){
  const dv = displayValue();
  stars.forEach((star, idx) => {
    const i = idx + 1;
    const portion = Math.min(1, Math.max(0, dv - (i - 1)));  // 0 … 1
    star.querySelector(".star-fill").style.width = (portion * 100) + "%";
  });
  readout.textContent = dv.toFixed(step === 0.5 ? 1 : 0);
}
    
      
    4. 
      
  4. 
        
    4
    Hover preview + which half of the star
    
        
    On mousemove over a star, measure where the pointer sits relative to that star's box. If half-stars are on and the pointer is in the left half, the preview is i − 0.5; otherwise it is i. Store it in hover and re-render — value is untouched. When the pointer leaves the whole strip, clear hover back to null and the committed value snaps back into view.
    
        
    star.addEventListener("mousemove", e => {
  const r = star.getBoundingClientRect();
  const leftHalf = step === 0.5 && (e.clientX - r.left) < r.width / 2;
  hover = i - (leftHalf ? 0.5 : 0);
  render();
});
strip.addEventListener("mouseleave", () => { hover = null; render(); });
    
      
    5. 
      
  5. 
        
    5
    Click to commit (and click-again to clear)
    
        
    A click is the moment the preview becomes real: copy whatever hover is showing into value. A nice touch is a toggle — clicking the exact value you already have clears it back to zero, which gives people an obvious way to undo a mis-tap without hunting for a separate button.
    
        
    star.addEventListener("click", () => {
  const picked = hover !== null ? hover : i;
  value = (picked === value) ? 0 : picked;   // click same star again → clear
  announce();                                // tell assistive tech
  render();
});
    
      
    6. 
      
  6. 
        
    6
    Keyboard: it is really a slider
    
        
    A rating is a value on a bounded scale, which is exactly a slider. Make the strip one focusable element and adopt the slider key model: →/↑ add a step, ←/↓ subtract, Home goes to 0 and End to MAX. The step is 0.5 or 1 depending on the half-star mode, so the same keys give whole or half increments for free. Keyboard changes commit immediately, so clear hover first.
    
        
    strip.addEventListener("keydown", e => {
  const map = { ArrowRight:+step, ArrowUp:+step, ArrowLeft:-step, ArrowDown:-step };
  let v = value;
  if (e.key in map)      v = value + map[e.key];
  else if (e.key==="Home") v = 0;
  else if (e.key==="End")  v = MAX;
  else return;                                  // ignore other keys
  e.preventDefault();
  hover = null;                                 // committing, not previewing
  value = Math.min(MAX, Math.max(0, v));
  announce(); render();
});
    
      
    7. 
      
  7. 
        
    7
    ARIA: expose it as a slider with a spoken value
    
        
    Give the strip role="slider" with aria-valuemin, aria-valuemax and a live aria-valuenow. The important extra is aria-valuetext: a screen reader announcing "3.5" is meaningless, but "3.5 out of 5 stars, Good" is clear. Update both on every commit, and never signal the rating by gold colour alone — the numeric read-out and the value-text carry it for colour-blind and non-visual users.
    
        
    strip.setAttribute("role", "slider");
strip.setAttribute("aria-valuemin", 0);
strip.setAttribute("aria-valuemax", MAX);

function announce(){
  const txt = value ? `${value} out of ${MAX} stars, ${labelFor(value)}` : "No rating";
  strip.setAttribute("aria-valuenow", value);
  strip.setAttribute("aria-valuetext", txt);
  liveRegion.textContent = txt;               // aria-live="polite" mirror
}
    
      
    8. 
      
  8. 
        
    8
    Modes: half-stars, read-only, reset
    
        
    Two switches cover most real needs. Toggling half-stars just flips step between 0.5 and 1, snapping any existing value to a whole number when you turn precision off. A read-only mode (for showing an average score rather than collecting one) removes the pointer cursor, drops the strip out of the tab order and ignores input — but keeps the exact same rendering, so a 3.7 average still displays correctly. Reset simply sets value = 0.
    
        
    function setHalf(on){ step = on ? 0.5 : 1; if(!on) value = Math.round(value); render(); }

function setReadOnly(on){
  readOnly = on;
  strip.tabIndex = on ? -1 : 0;               // out of tab order when display-only
  strip.setAttribute("aria-readonly", on);
  strip.classList.toggle("readonly", on);
}
resetBtn.onclick = () => { value = 0; hover = null; announce(); render(); };
// mousemove / click / keydown all early-return while readOnly is true.
