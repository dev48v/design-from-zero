🔎 Autocomplete / Combobox — Accessible Typeahead

Start typing a country — the list filters as you type.

This is a real combobox over ~200 countries. As you type it does a case-insensitive substring match, highlights the matched letters, and drops a suggestions listbox below the input. Everything works from the keyboard: ↓/↑ move the active option (wrapping around the ends), Enter selects it, Esc closes, Tab moves on. Click a row to pick it, click anywhere outside to close, and the filter is debounced (~120ms) so it doesn't run on every single keystroke. When nothing matches you get a friendly empty state — no dead-end.

▼
    Matches: 200
    Active: none
    Selected: —

    The input is role="combobox" with aria-expanded and aria-activedescendant; the dropdown is a role="listbox" of role="option" items. The active option (keyboard highlight) is tracked with aria-activedescendant so focus never leaves the input, while aria-selected marks the visually-highlighted row. A polite live region announces the match count for screen readers. This is the WAI-ARIA "editable combobox with list autocomplete" pattern, built in ~110 lines of vanilla JS.

    🔨 Build it yourself
    
    
    A text input, a debounced filter over an array, a rendered listbox, keyboard navigation with a tracked active index, selection, close-on-outside-click, and the ARIA combobox wiring. ~110 lines of vanilla JS — no framework, no dependencies.
    
    
      
      
    1. 
        
      1
      Markup: a combobox, not a <datalist>
      
        
      A plain text input plus a <ul role="listbox"> you control yourself. The input gets role="combobox", aria-expanded, aria-autocomplete="list" and aria-controls pointing at the list. This gives you full control over styling, filtering and highlighting — none of which native <datalist> lets you touch.
      
        
      <div class="ac-wrap">
  <input id="acInput" role="combobox" aria-expanded="false"
         aria-autocomplete="list" aria-controls="acList" autocomplete="off">
  <ul id="acList" role="listbox" hidden></ul>
</div>
      
      
      2. 
      
    2. 
        
      2
      Debounce the filter
      
        
      Filtering on every keystroke is wasteful — a fast typist fires 8–10 input events a second. A tiny debounce collapses a burst of keystrokes into one filter run after the user pauses ~120ms. For a local array it's mostly a nicety; for a remote API (Step 9) it's what stops you hammering the server with a request per letter.
      
        
      function debounce(fn, ms){
  let t; return (...a) => { clearTimeout(t); t = setTimeout(() => fn(...a), ms); };
}
input.addEventListener("input", debounce(onType, 120));
      
      
      3. 
      
    3. 
        
      3
      Filter + highlight the match
      
        
      Lower-case both the query and each candidate, keep the ones whose name contains the query (substring match), and wrap the matched span in <mark> so it stands out. An empty query shows the full list. Render the survivors into the listbox; if there are none, render an empty state instead of a blank box.
      
        
      function filter(q){
  q = q.trim().toLowerCase();
  if (!q) return COUNTRIES;
  return COUNTRIES.filter(c => c.name.toLowerCase().includes(q));
}
function highlight(name, q){
  const i = name.toLowerCase().indexOf(q.toLowerCase());
  if (i < 0 || !q) return name;
  return name.slice(0,i) + "<mark>" + name.slice(i,i+q.length) + "</mark>" + name.slice(i+q.length);
}
      
      
      4. 
      
    4. 
        
      4
      Render options with stable ids
      
        
      Each option is a <li role="option"> with a unique id (e.g. opt-3) so aria-activedescendant can point at it. Give it a data-index so a click knows which item it is. Open the list by unhiding it and flipping aria-expanded="true" on the input.
      
        
      list.innerHTML = results.map((c, i) =>
  `<li id="opt-${i}" role="option" data-index="${i}" aria-selected="false" class="ac-opt">
     <span>${highlight(c.name, q)}</span><span class="flag">${c.flag}</span>
   </li>`).join("");
list.hidden = false;
input.setAttribute("aria-expanded", "true");
      
      
      5. 
      
    5. 
        
      5
      Keyboard nav with a tracked active index
      
        
      Hold one number — active — the index of the highlighted option. ↓ does (active+1) % n and ↑ does (active-1+n) % n so it wraps at the ends. Call preventDefault() on the arrows so the caret doesn't jump. After moving, paint the row and scroll it into view.
      
        
      input.addEventListener("keydown", e => {
  const n = results.length; if (!n) return;
  if (e.key === "ArrowDown"){ e.preventDefault(); setActive((active + 1) % n); }
  else if (e.key === "ArrowUp"){ e.preventDefault(); setActive((active - 1 + n) % n); }
});
function setActive(i){
  active = i;
  [...list.children].forEach((li, idx) => li.setAttribute("aria-selected", idx === i));
  input.setAttribute("aria-activedescendant", "opt-" + i);
  list.children[i].scrollIntoView({ block: "nearest" });
}
      
      
      6. 
      
    6. 
        
      6
      Select — Enter, click, Esc
      
        
      Enter commits the active option; a click commits the clicked one (use mousedown so it fires before the input blurs). Selecting writes the value into the input and closes the list. Esc closes without choosing. Closing means hide the list, reset active = -1, clear aria-activedescendant and set aria-expanded="false".
      
        
      if (e.key === "Enter" && active > -1){ e.preventDefault(); select(active); }
if (e.key === "Escape") close();
list.addEventListener("mousedown", e => {
  const li = e.target.closest("[data-index]"); if (li) select(+li.dataset.index);
});
function select(i){ input.value = results[i].name; close(); }
function close(){ list.hidden = true; active = -1;
  input.setAttribute("aria-expanded","false"); input.removeAttribute("aria-activedescendant"); }
      
      
      7. 
      
    7. 
        
      7
      Close on outside click + announce
      
        
      A document-level pointerdown listener closes the list whenever the click lands outside the wrapper. Guard it with wrap.contains(e.target) so clicks inside are ignored. Finally, drop the match count into an aria-live="polite" region so screen-reader users hear "12 results" without the announcement stealing focus.
      
        
      document.addEventListener("pointerdown", e => {
  if (!wrap.contains(e.target)) close();
});
live.textContent = results.length + " result" + (results.length === 1 ? "" : "s");
      
      
      8. 
      
    8. 
        
      8
      Remote data (bonus)
      
        
      Swap the local filter() for a fetch() and the debounce earns its keep — you fire one request per pause, not per letter. Track a request id (or an AbortController) so a slow earlier response can't overwrite a fresh one — the classic out-of-order race. Show a loading row while it's in flight. The keyboard, ARIA and selection code above don't change at all.
      
        
      let reqId = 0;
async function remote(q){
  const my = ++reqId;
  const res = await fetch(`/api/search?q=${encodeURIComponent(q)}`);
  const data = await res.json();
  if (my !== reqId) return;        // a newer request already won — drop this
  render(data, q);
}
