🎯 Popover — An Anchored Floating Panel With Flip + Shift Positioning, a Tracking Arrow, Click / Hover / Focus Triggers, Dismiss & A11y

Anchor a real floating panel to any element.

A working popover built from scratch in vanilla JS — a tiny floating-ui. Pick a preferred side and alignment, then open a panel on any anchor below. The math reads the anchor's rectangle, measures the panel, and places it on your chosen side — but if that side would spill off-screen it flips to the opposite side, and it shifts along the edge to stay inside the viewport. The arrow always points back at the anchor's centre and tracks that shift. Switch the trigger to hover or focus, tab into the fields inside, and dismiss with an outside click or Esc. No library.

This is the engine behind every rich tooltip, menu, date-picker and hover-card. Unlike a modal drawer it does not take over the screen — it stays pinned to the thing that opened it and only nudges itself to stay visible. Real production apps reach for floating-ui for the edge cases (scroll containers, iframes, transforms), but the core is exactly the flip + shift + arrow math you can drive here.

🔨 Build it yourself

    
A popover is a position:fixed panel whose coordinates you compute yourself from the anchor's rectangle. Place it on a preferred side, flip to the opposite side when it would overflow, shift it along the edge to stay in the viewport, and point an arrow at the anchor that tracks the shift. Then wire click / hover / focus triggers, outside-click + Esc dismissal, focus management, and a reposition on scroll/resize. ~120 lines of vanilla JS, no dependencies.

    
    
      
  1. 
        
    1
    Read the anchor rect, float the panel
    
        
    The anchor is any element on the page; the panel floats above everything at position:fixed. getBoundingClientRect() hands you the anchor's box in viewport coordinates — exactly the space a fixed element lives in — so its numbers drop straight into the panel's top/left. Measure the panel with offsetWidth/offsetHeight while it is rendered (not display:none).
    
        
    .pop { position:fixed; z-index:70; visibility:hidden; }   /* laid out, measurable */
.pop.open { visibility:visible; }

const a  = anchor.getBoundingClientRect();      // {top,left,width,height} in viewport space
const pw = pop.offsetWidth, ph = pop.offsetHeight;
const vw = innerWidth,      vh = innerHeight;
    
      
    2. 
      
  2. 
        
    2
    Place it on the preferred side + alignment
    
        
    Two axes. The side (top/bottom/left/right) sets the main-axis coordinate: a bottom popover sits at the anchor's bottom edge plus a small gap. The alignment (start/center/end) sets the cross-axis: center lines the panel's midpoint up with the anchor's. That is the whole "preferred" placement, before any correction.
    
        
    const GAP = 12;
let top, left;
if (side === "bottom") top = a.top + a.height + GAP;   // main axis
if (side === "top")    top = a.top - GAP - ph;

// cross axis (for a vertical side, that's horizontal):
if (align === "start")  left = a.left;
if (align === "center") left = a.left + a.width/2 - pw/2;
if (align === "end")    left = a.left + a.width - pw;
    
      
    3. 
      
  3. 
        
    3
    Flip when the preferred side overflows
    
        
    Before committing, ask: does the panel actually fit on that side? A bottom popover fits only if anchor.bottom + gap + panelHeight is still above the viewport floor. If it does not fit but the opposite side does, flip. This is why a menu near the bottom of the screen opens upward instead of getting clipped.
    
        
    const opposite = { top:"bottom", bottom:"top", left:"right", right:"left" };
function fits(s){
  if (s === "bottom") return a.top + a.height + GAP + ph <= vh - PAD;
  if (s === "top")    return a.top - GAP - ph >= PAD;
  if (s === "right")  return a.left + a.width + GAP + pw <= vw - PAD;
  if (s === "left")   return a.left - GAP - pw >= PAD;
}
if (!fits(side) && fits(opposite[side])) side = opposite[side];   // flip
    
      
    4. 
      
  4. 
        
    4
    Shift along the edge to stay in view
    
        
    Flipping fixes the main axis; shifting fixes the cross axis. A bottom popover aligned to a left-edge anchor can still poke off the left of the screen. Clamp its cross-axis coordinate between the viewport padding and viewport − panel − padding. Remember how far you moved — the arrow will need it.
    
        
    const vertical = side === "top" || side === "bottom";
let shift = 0;
if (vertical){                                   // clamp horizontally
  const min = PAD, max = vw - pw - PAD;
  const clamped = Math.max(min, Math.min(left, max));
  shift = clamped - left;                        // how far we nudged it
  left  = clamped;
}                                                // (mirror for horizontal sides)
    
      
    5. 
      
  5. 
        
    5
    The arrow — point at the anchor, track the shift
    
        
    The arrow must point at the anchor's centre, not the panel's. Compute the anchor centre in the cross axis, subtract the panel's edge to get an offset inside the panel — this naturally accounts for the shift. Then clamp it so a heavy shift never pushes the arrow past the panel's rounded corner.
    
        
    const AM = 16;                                   // keep arrow off the corners
let arrow;
if (vertical){
  const centerX = a.left + a.width/2;            // anchor centre
  arrow = centerX - left;                        // offset within the panel
  arrow = Math.max(AM, Math.min(arrow, pw - AM));// clamp to panel edges
}
// arrow is a CSS border-triangle nudged half its base: left = arrow - 8
    
      
    6. 
      
  6. 
        
    6
    Triggers — click, hover, focus
    
        
    One openFor(anchor) / close() pair, three ways to fire it. Click toggles (and closes on outside click). Hover opens on mouseenter and closes on mouseleave — but with a small delay and a listener on the panel too, so the mouse can travel from anchor to panel without it snapping shut. Focus opens on focus / closes on blur for keyboard users.
    
        
    anchor.addEventListener("click", () => {
  if (mode === "click") current === anchor ? close() : openFor(anchor);
});
anchor.addEventListener("mouseenter", () => { if (mode === "hover"){ clearTimeout(t); openFor(anchor); }});
anchor.addEventListener("mouseleave", () => { if (mode === "hover") t = setTimeout(close, 120); });
pop.addEventListener("mouseenter",    () => clearTimeout(t));        // bridge the gap
anchor.addEventListener("focus", () => { if (mode === "focus") openFor(anchor); });
    
      
    7. 
      
  7. 
        
    7
    Dismiss + focus management
    
        
    Because a popover can hold interactive content (a form, buttons), it is more than a tooltip. On a click open, move focus into the panel and give it role="dialog" with an aria-labelledby; the anchor gets aria-expanded. Dismiss on outside mousedown and on Esc, and on close return focus to the anchor so the keyboard user is never stranded.
    
        
    document.addEventListener("mousedown", e => {
  if (current && !pop.contains(e.target) && !isAnchor(e.target)) close();   // outside click
});
document.addEventListener("keydown", e => {
  if (e.key === "Escape" && current){ const a = current; close(); a.focus(); }
});
function openFor(a){ current = a; pop.classList.add("open");
  a.setAttribute("aria-expanded","true"); if (mode==="click") firstField().focus(); }

  8. 
        
    8
    Reposition live + the native platform note
    
        
    A fixed panel is placed once — but the anchor moves when the page scrolls or the window resizes, so recompute on both (use capture-phase scroll to catch nested scrollers). The platform is catching up: the native Popover API (popovertarget) gives you top-layer + light-dismiss for free, and CSS anchor-positioning (anchor() / position-try) does flip + shift in pure CSS. Support is still uneven, which is exactly why floating-ui exists.
    
        
    addEventListener("scroll", () => current && reposition(), true);   // capture nested scroll
addEventListener("resize", () => current && reposition());

<!-- native, no JS positioning yet everywhere: -->
<button popovertarget="p">Open</button>
<div id="p" popover>…</div>
